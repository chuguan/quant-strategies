## 版本号规则

```
V260529-09
│ │     └─ 子策略迭代号（09=真实涨日第9版优化）
│ └─────── 主版本日期（2026年5月29日定版）
└───────── 版本前缀
```

- 主版本号（V2605xx）：整体重构时升
- 子版本号（-09）：单个行情评分有实质性突破时升
- 无突破不升号

## 迭代工作流

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ master01 │ →  │ master02 │ →  │ master03 │ →  │ release  │ →  │  生产     │
│ (小改1)   │    │ (小改2)   │    │ (小改3)   │    │ (验收通过)│    │ (定时器)  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
   dev/current     打快照         打快照          升release       部署生产
```

### 详细步骤

#### 1️⃣ 开发（在 dev/current 里改）
```bash
cd dev/current/
# 改子策略文件，更新文件头版本号（如 V260529-10 → V260529-11）
```

#### 2️⃣ 打master快照（每次小迭代）
```bash
# 从 dev/current 打快照
cp dev/current/* dev/snapshots/master02/
```

#### 3️⃣ 测试
```bash
python dev/current/大道至简_统一测试.py
```

#### 4️⃣ 升release（验收通过后）
```bash
# 创建新release目录
mkdir -p release/V260530/

# 把dev/current的内容复制过去
cp dev/current/* release/V260530/
```

#### 5️⃣ 部署生产
```bash
# 先存档旧生产版本
mkdir -p archive/V260529/
cp 大道至简_V260529.py archive/V260529/
cp 大道至简_*_评分策略.py archive/V260529/

# 把新release覆盖到生产根目录
cp release/V260530/大道至简_V260529.py ./
cp release/V260530/大道至简_*_评分策略.py ./

# 更新定时器（如果需要）
# cronjob action=update job_id=xxx script=大道至简_V260530.py

# 更新本版本管理文件
```
